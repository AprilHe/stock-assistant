# Planning Docs

Read documents in this order:

1. `v2.md` — canonical implementation plan for the current system
2. `v1.md` — archived original planning context

## Why two files exist
- `v1.md` captured the original single-analysis architecture and delivery setup
- `v2.md` upgrades that design into a portfolio-aware decision engine

## Rule for future work
- Update `v2.md` for active planning
- Do not add `v3.md`, `final-v2.md`, or parallel planning files
- If major direction changes again, either:
  - replace `v2.md` and record a short changelog section inside it, or
  - move the old file into an `archive/` folder and keep one canonical active plan
