### What Each Data Source Does

**yfinance** — answers "what happened to prices?"
- Fetches current price, % change, volume for any ticker
- Free, no API key, covers stocks + commodities + crypto
- Example output: `{"S&P 500": {"price": 5123.41, "change_pct": -1.2}}`

**NewsAPI** — answers "why did prices move?"
- Fetches recent headlines: "Fed signals rates stay high", "Tech selloff continues"
- Without news, the LLM can only say "it dropped" — not "it dropped because..."
- Free tier: 100 requests/day, plenty for personal use
- Alternative: **Finnhub** (covers both price + news, more financial-specific data)

**LLM (Claude / GPT / Gemini)** — answers "what does this mean for me?"
- Takes the combined price data + news as input (called a "prompt")
- Outputs a human-readable summary and investment commentary
- This is the "brain" — it synthesizes everything into actionable insight
- Completely swappable: same code, different provider

### LLM Comparison for This Use Case

| Provider | Model | Strengths | Free Tier | Est. Cost/Analysis |
|---|---|---|---|---|
| Anthropic | claude-sonnet-4-6 | Deep reasoning, nuanced analysis | No | ~$0.01 |
| OpenAI | gpt-4o | Most widely tested, reliable | No | ~$0.02 |
| Google | gemini-1.5-flash | Large free quota, fast | Yes (generous) | Free / very low |
| Groq | llama-3.3-70b | Extremely fast, open source model | Yes | Free / very low |

**Recommendation for getting started:** Use **Gemini** (free) to build and test, then switch to Claude or GPT-4o for better analysis quality in production.

## Tech Stack

| Layer | Technology | Role |
|---|---|---|
| Language | Python 3.11+ | Everything runs in Python |
| Market Prices | `yfinance` | Fetches price + % change data (free, no key) |
| Financial News | `NewsAPI` | Fetches headlines that explain price movements |
| LLM Abstraction | `LiteLLM` | One interface to call ANY LLM provider |
| LLM Provider | Claude / GPT-4o / Gemini | Turns data into readable analysis |
| Notifications | Telegram Bot API | Sends weekly reports to your phone |
| Web Backend | FastAPI + `aiofiles` | Serves the API and web dashboard (aiofiles needed for static file serving) |
| Web Frontend | Plain HTML + Tailwind CSS (CDN) | Dashboard UI, no build tools needed |
| Scheduler | APScheduler 3.x (pin `apscheduler==3.10.4`) | Runs weekly jobs automatically — v3/v4 APIs differ significantly |
| Config | `.env` + `python-dotenv` | Keeps all API keys out of code |

## Why LiteLLM? (Key Architecture Decision)

Without LiteLLM, switching providers means rewriting code:
```python
# Without LiteLLM — tightly coupled, painful to change
import anthropic
client = anthropic.Anthropic()
response = client.messages.create(model="claude-sonnet-4-6", ...)

# To switch to OpenAI you'd have to rewrite everything
import openai
client = openai.OpenAI()
response = client.chat.completions.create(model="gpt-4o", ...)
```

With LiteLLM — change one line to switch providers:
```python
# With LiteLLM — clean abstraction
from litellm import completion

# Switch provider by changing just the model string:
response = completion(model="claude-sonnet-4-6", messages=[...])   # Anthropic
response = completion(model="gpt-4o", messages=[...])              # OpenAI
response = completion(model="gemini/gemini-1.5-flash", messages=[...])  # Google
response = completion(model="groq/llama-3.3-70b-versatile", messages=[...])  # Groq
```

The `LLM_MODEL` is stored in `.env`, so you switch providers without touching any code.

## Folder Structure

```
stock-assistant/
├── .env                    # Your actual API keys (never share or commit)
├── .env.example            # Template — safe to share/commit
├── .gitignore              # Excludes .env and other secrets from git
├── requirements.txt
├── README.md
├── main.py                 # FastAPI app + server startup
│
├── core/
│   ├── __init__.py         # Makes core/ a Python package
│   ├── market_data.py      # yfinance: fetch prices for watchlist
│   ├── news.py             # NewsAPI: fetch headlines by topic/ticker (with 1hr cache)
│   ├── ai_analysis.py      # LiteLLM: build prompt + call LLM
│   └── scheduler.py        # APScheduler: weekly Telegram job
│
├── telegram_bot/
│   ├── __init__.py         # Makes telegram_bot/ a Python package
│   └── bot.py              # Telegram bot commands (runs in background thread)
│
└── web/
    └── index.html          # Dashboard frontend
```

> **Note on Telegram + FastAPI:** `python-telegram-bot` v20+ is async and uses its own event loop. Run the bot via `application.run_polling()` in a background thread started at FastAPI startup. Do not try to run both in the same event loop.

## Build Order — Step-by-Step Prompts for Claude Code

### Step 1: Project Setup
```
Create a Python project called "stock-assistant" with the exact folder 
structure in my planning doc. Create requirements.txt with: fastapi, uvicorn, 
yfinance, litellm, python-dotenv, requests, apscheduler, python-telegram-bot.
Create a .env.example file matching the template in the planning doc.
Create a basic main.py that starts a FastAPI server on port 8000.
```

### Step 2: Market Data Module
```
In core/market_data.py, write a function get_market_snapshot() that uses
yfinance to fetch today's price and % change for all tickers in the WATCHLIST
dict defined in the planning doc. Handle errors gracefully if a ticker fails.
Return a clean dictionary. Add a __main__ block to test and print the result.

IMPORTANT: yfinance returns empty DataFrames silently (bad ticker, market
holiday, network issue). Always check `if df.empty` and return a fallback
value (e.g. {"price": None, "change_pct": None, "error": "data unavailable"})
rather than crashing.
```

### Step 3: News Module
```
In core/news.py, write a function get_market_news(query="stock market",
num_articles=5) that calls NewsAPI to fetch recent financial headlines.
Return a list of {title, description, url} dicts. Load NEWS_API_KEY from .env.
Add a __main__ block to test it.

IMPORTANT: Cache responses for 1 hour (use a simple dict with timestamp).
NewsAPI free tier is 100 req/day — caching prevents quota exhaustion when
multiple users call analyze_ticker() or the scheduler fires near a manual call.
```

### Step 4: AI Analysis Module (with LiteLLM)
```
In core/ai_analysis.py, write two functions using LiteLLM:

1. generate_market_summary(market_data: dict, news: list) -> str
   Builds a prompt combining price data + news headlines.
   Calls LiteLLM using the LLM_MODEL from .env.
   Returns a 200-word market summary with investment commentary.
   End every response with this disclaimer:
   "⚠️ For informational purposes only. Not financial advice."

2. analyze_ticker(ticker: str, company_name: str) -> str  
   Fetches 1-month price history for the ticker via yfinance.
   Also fetches news about the company via NewsAPI.
   Returns a focused 150-word analysis of that specific stock.

Load all API keys from .env via python-dotenv.
```

### Step 5: Telegram Bot
```
In telegram_bot/bot.py, create a Telegram bot with these commands:

/start — welcome message explaining the bot
/summary — fetches market snapshot + news, generates AI summary, sends it
/analyze TICKER — e.g. /analyze META — runs analyze_ticker() and sends result

Use the functions from core/market_data.py, core/news.py, core/ai_analysis.py.
Load TELEGRAM_BOT_TOKEN and TELEGRAM_CHAT_ID from .env.
```

### Step 6: Scheduler
```
In core/scheduler.py, use APScheduler to create a job that:
- Runs every Monday at 8:00 AM (local time)
- Calls the full pipeline: get_market_snapshot + get_market_news + 
  generate_market_summary
- Sends the result to the Telegram chat

Wire the scheduler to start when the FastAPI app starts in main.py.
```

### Step 7: Web Dashboard
```
Add these routes to main.py:
- GET / → serve web/index.html  
- GET /api/summary → return market summary as JSON
- GET /api/analyze/{ticker} → return AI analysis for any ticker as JSON

In web/index.html, create a clean single-page dashboard using Tailwind CSS 
(CDN only, no build tools). It should show:
1. A market overview card with current prices
2. The AI weekly summary
3. A search box: user types a ticker (e.g. META) and gets AI analysis below
All text in English. Mobile-friendly layout.
```
